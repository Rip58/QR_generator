# QR Code Generator App - Design Document

## 1. Overview
A modern, simple, and aesthetic web application to generate, customize, download, and save QR codes. The focus is on a premium "Capsule" design aesthetic, privacy (local storage), and ease of use.

## 2. Design System & UX
**Inspiration:** "Domio" / Capsule Style (from user images).
**Theme:** High contrast, rounded shapes, fluid interactions.

### Color Palette
- **Primary Cyan:** `#00FFD4` (Accents, Highlights)
- **Primary Blue:** `#5465FF` (Buttons, Active States)
- **Soft Lavender:** `#D2DDFF` (Backgrounds, Secondary Elements)
- **Deep Black:** `#050208` (Text, Strong UI Elements, Dark Mode Background)
- **White:** `#FFFFFF` (Cards, Text on Dark)

### Typography
- **Primary Font:** Modern Sans-Serif (e.g., 'Inter' or 'Outfit' as recommended in web_application_development).

### UI Components (Capsule Style)
- **Buttons:** Fully rounded "pill" shapes. High contrast (e.g., White button with Black icon circle).
- **Cards:** Large border-radius (e.g., `rounded-3xl`), clean padding.
- **Inputs:** Floating labels or clean underlined/capsule inputs.
- **Navigation:**
    - Top Bar: Branding (Left), "Page Indicator" Icon (Right) as requested.
    - Menu: Hamburger menu triggering a smooth overlay or sidebar.

## 3. User Flow
1.  **Landing:** Clean interface to input content (link, instagram, etc.).
2.  **Generation:** Real-time preview of the QR code as user types/configures.
3.  **Customization:** Controls to change QR foreground/background colors using the palette.
4.  **Action:**
    - **Download:** Save as PNG/SVG.
    - **Save:** Add to "History".
5.  **History View:** List of previously generated QRs (stored locally). Clicking one restores the state.

## 4. Architecture

### Tech Stack
- **Framework:** Next.js (React)
- **Styling:** Tailwind CSS (configured with custom typography and palette)
- **Icons:** Lucide React (clean, standard)
- **QR Library:** `qrcode.react` (Robust, supports rich customization)

### Data Persistence
- **Storage:** `localStorage` (Privacy focused, no backend required).
- **Structure:**
    ```json
    {
      "id": "uuid",
      "createdAt": "timestamp",
      "type": "link|instagram|wifi",
      "content": "https://...",
      "settings": { "fgColor": "#...", "bgColor": "#..." }
    }
    ```

## 5. Components Structure
- `Layout`: Handles the persistent Top Bar and Navigation state.
- `QRGenerator`: Main workspace.
    - `InputSection`: Tabs for Link / Social / Text.
    - `PreviewSection`: The QR canvas + Download button.
    - `StyleControls`: Color pickers and shape sliders.
- `HistoryDrawer`: A slide-over or graphical list of saved codes.
- `CapsuleButton`: Reusable styled button component.

## 6. Verification
- **Visual:** Does it match the "Capsule" aesthetic?
- **Functional:** Do QRs scan correctly? (Manual test with phone).
- **Persistence:** Does history survive page reload?
